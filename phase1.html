Parcourir le web

Le but du premier travail est d'écrire un programme qui parcourt des pages web en traversant des hyperliens tout en :

  • Sauvegardant localement les pages explorées
  • Collectant les adresses courriels trouvées dans les pages

Correction

L'ensemble de la correction est effectuée sur le site http://5a5.di.college-em.info/testbot/. Vous pouvez tester votre bot sur d'autres sites mais ça n'aura pas d'impact sur la correction.

Gestion des arguments

Arguments en ligne de commandes

Votre application doit prendre 3 arguments en ligne de commande dans l'ordre spécifié ici.

Attention! Il ne s'agit pas d'une application intéractive. Vous devez passer les paramètres en ligne de commandes. Il est fortement recommander de créer plusieurs configurations d'exécution dans Intellij IDEA.

1 point Dans le cas où l'utilisateur ne fournit pas le bon nombre d'arguments vous devez lui indiquer clairement comment utiliser votre application. (Combien d'arguments, quel est le rôle de chacun).

  1. 2 points la profondeur d'exploration qui doit être un nombre entier positif.
  2. 2 points l'url de départ qui est la première page à explorer. Il doit s'agir d'une adresse valide (le format est correct et il y a une page qui y correspond).
  3. 2 points le répertoire où écrire les copies locales des fichiers explorés. Le dossier doit être accessible et on doit pouvoir y écrire.

Pour chaque arguement, vous obtenez un point s'il est bien pris en compte et un point si vous affichez un message significatif si la valeur n'est pas bonne.

Traitement

Gestion de la profondeur

3 points Le paramètre de profondeur d'exploration fonctionne ainsi:

  1. Une profondeur de 0 signifie qu'on explore uniquement la page de départ fournie en argument
  2. Une profondeur de 1 signifie qu'on explore la page initiale, puis les liens contenus dans la page initiale
  3. On explore la page initiale, puis les liens de cette page, puis les liens contenus dans les pages liées
  4. etc.

Mémoire de l'exploration

2 points Vous devez implanter un mécanisme pour éviter de visiter plusieurs fois la même page. Si une page A contient un lien vers B qui contient un lien vers A, on ne devrait pas re-explorer la page A.

Vous pouvez tester le bon fonctionnement en lançant votre programme avec une très grande profondeur d'exploration (ex. 10000).

Messages pendant l'exploration

1 point Vous devez fournir des messages significatifs dans la console pour chaque exploration d'une page web contenant l'URL complète du fichier.

1 point Dans le cas où une URL ne peut être explorée, vous devez fournir un message significatif incluant l'URL et le problème

Sauvegarde des fichiers explorés

1 point Chaque page html explorée doit être stockée dans un fichier avec le même nom que la resource explorée (index.html, 1.html etc.).

1 point Chaque apparition d'une adresse courriel dans le contenu d'une page doit être remplacé par votre adresse courriel avant l'écriture sur le disque.

2 points Les liens relatifs entre pages téléchargées doivent être fonctionnels et doivent pointer vers les fichiers locaux si 2 pages sont dans le même dossier.

2 points Les liens relatifs doivent fonctionner entre resources locales dans des dossiers différents.

Collecte des courriels

1 point Vous devez collecter les adresses courriels contenues dans chaque page pendant l'exploration.

1 point À la fin de l'exécution, vous devez afficher dans la console la liste des courriels collectés sans doublon dans l'ordre alphabétique.
